# Chopin Framework — Improvement Plan



Based on the current codebase state (March 2026).  Covers all crates: `chopin-core`, `chopin-pg`, `chopin-orm`, `chopin-auth`, `chopin-cli`.

Legend: ✅ done · 🔧 partial · ❌ not started

## Shared-Nothing TPC Architecture — Status

Chopin's thread-per-core model is the foundation for eliminating OS scheduling overhead
(context switches, cache-line bouncing).  Current status of the four pillars:

| Pillar | Status | Location | Notes |
|--------|--------|----------|-------|
| `SO_REUSEPORT` socket sharding | ✅ | `syscalls::create_listen_socket_reuseport()` | Each worker owns its own listener — zero cross-thread accept contention |
| CPU pinning (core affinity) | ✅ | `server.rs` → `core_affinity::set_for_current()` | One thread per `num_cpus::get()`, pinned at spawn |
| Thread-per-core event loops | ✅ | `worker.rs` → `run_epoll()` / `run_uring()` | Connection accepted, processed, and closed on the same core |
| Spin-poll / zero-timeout epoll | ✅ | `worker.rs` | `CHOPIN_EPOLL_TIMEOUT_MS=0` for spin-poll — see Sprint D.8 |
| io_uring SQPOLL | ✅ | `worker.rs` | `CHOPIN_SQPOLL=1` enables kernel SQ polling — see Sprint A.3 |

## Sprint A — io_uring Hardening (chopin-core) — HIGH PRIORITY

The io_uring backend already lives in `worker.rs` behind `#[cfg(all(target_os = "linux", feature = "io-uring"))]`
guards (~750 lines, 17 conditional blocks).  `run_uring()` dispatches from `Worker::run()`, multishot
accept works, and the full read → parse → serialize → writev pipeline is operational.  Remaining
work is performance hardening and CI integration.

| # | Item | Status | Notes |
|---|------|--------|-------|
| A.1 | io_uring event loop in `worker.rs` (single-file, `#[cfg]` guards) | ✅ | Rust idiomatic — shared parse/slab/serialize stays unified |
| A.2 | Multi-shot accept via `prep_accept_multishot` (kernel ≥5.19) | ✅ | `submit_accept()` already uses `IORING_ACCEPT_MULTISHOT` |
| A.3 | SQPOLL mode — opt-in via `IORING_SETUP_SQPOLL` flag | ✅ | `CHOPIN_SQPOLL=1` env var enables kernel-side SQ polling |
| A.4 | Fixed / registered buffer rings (`io_uring_register`) | ✅ | `register_buffers()` in `run_uring()` pins slab read/write buffers in kernel memory |
| A.5 | Sendfile via io_uring for `Body::File` (fallback to sync today) | ✅ | `prep_splice()` + `handle_splice()` integrated; zero-copy file transfer via io_uring splice |
| A.6 | `Worker::run_uring()` wired into `Worker::run()` dispatch | ✅ | `run()` branches on `cfg` at worker.rs:106 |
| A.7 | Benchmark `io-uring` vs epoll on Linux (`wrk -t16 -c1024`) | ✅ | `bench_uring_vs_epoll.sh` script with configurable concurrency |
| A.8 | CI: run io-uring feature tests only on `target_os = "linux"` | ✅ | `test-iouring` CI job on ubuntu-latest with `--features io-uring` |

## Sprint B — chopin-pg: TLS/SSL & Auth Gaps — HIGH PRIORITY

TLS is a **production blocker** — no cloud-hosted PostgreSQL (RDS, Neon, Supabase, Cloud SQL)
accepts plain-text connections by default.

| # | Item | Status | Notes |
|---|------|--------|-------|
| B.1 | TLS/SSL support — `rustls` behind `tls` feature flag | ✅ | Custom zero-dep TLS negotiation; `SslMode` enum; `TlsStream` wrapper |
| B.2 | `PgConfig::ssl_mode` field (disable / prefer / require / verify-ca) | ✅ | Parsed from URL `?sslmode=`; `SslMode` enum with `prefer`/`require` |
| B.3 | SCRAM-SHA-256-PLUS (channel binding over TLS) | ✅ | `p=tls-server-end-point` channel binding; SHA-256 cert hash in `tls.rs` |
| B.4 | MD5 password authentication | ✅ | RFC 1321 hash; handles `AuthenticationMD5Password` messages |
