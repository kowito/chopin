## Sprint A тАФ io_uring Worker (chopin-core) тАФ HIGH PRIORITY

The syscall layer (`syscalls::uring`) and `conn::Conn.pending_op` are already in place, but
`worker_uring.rs` does not yet exist.  This is the biggest remaining performance opportunity on Linux.

| # | Item | Status | Notes |
|---|------|--------|-------|
| A.1 | Create `src/worker_uring.rs` with full io_uring event loop | тЭМ | ~700 lines mirroring worker.rs |
| A.2 | Multi-shot accept via `prep_accept_multishot` (kernel тЙе5.19) | тЭМ | One SQE тЖТ many CQEs, no re-arm |
| A.3 | SQPOLL mode тАФ opt-in via `IORING_SETUP_SQPOLL` flag | тЭМ | Flag scaffolded; not wired to Worker |
| A.4 | Fixed / registered buffer rings (`io_uring_register`) | тЭМ | Eliminates per-read copy |
| A.5 | Sendfile via io_uring for `Body::File` (fallback to sync today) | тЭМ | |
| A.6 | `Worker::run_uring()` wired into `Worker::run()` dispatch | тЭМ | `run_epoll` branch exists; uring branch missing |
| A.7 | Benchmark `io-uring` vs epoll on Linux (`wrk -t16 -c1024`) | тЭМ | Needed to validate win |
| A.8 | CI: run io-uring feature tests only on `target_os = "linux"` | тЭМ | Guard with `cfg_attr` in workflow |
